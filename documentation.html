ConnectSlider – Documentation

Native JavaScript carousel/slider (no jQuery). Replaces bxslider with the same API subset and right-click safety.

Open example.html to see all use cases (blog carousel, group carousel, right-click test, minimal).

Overview

  • Horizontal carousel with configurable visible slides, slide width, margin, and move step.
  • Click and drag (or touch swipe) to scroll the slider; release to snap or advance to next/prev if drag exceeds threshold.
  • Optional prev/next controls; can be placed in external elements via nextSelector / prevSelector.
  • Optional pager (not shown in current examples).
  • Callbacks: onSliderLoad, onSlideBefore, onSlideAfter.
  • Right-click safety: Only left-clicks (and touch) trigger navigation. Right-click on controls or track does not move the slider or start auto.

Usage

Pass the list element (e.g. <ul>) and an options object. The returned instance is also stored on the element as element._connectSlider.

var listEl = document.querySelector('#my-slider');
var slider = ConnectSlider(listEl, {
  slideWidth: 335,
  slideMargin: 23,
  minSlides: 3,
  maxSlides: 4,
  moveSlides: 4,
  nextSelector: '#my-next',
  prevSelector: '#my-prev',
  hideControlOnEnd: true,
  infiniteLoop: false,
  onSliderLoad: function () { console.log('ready'); },
  onSlideAfter: function () { console.log('slide changed'); }
});

// Later: destroy or navigate
slider.goToNextSlide();
slider.destroySlider();

// Or get instance from element
var same = listEl._connectSlider;

Callbacks

  • onSliderLoad(currentIndex) – Fired once when the slider is ready. this is the list element.
  • onSlideBefore(slideElement, oldIndex, newIndex) – Before transition. Return false to cancel.
  • onSlideAfter(slideElement, oldIndex, newIndex) – After transition. this is the list element.

Right-click and accessibility

Only left-clicks (and touch, where button is undefined) trigger prev/next or pager. Right-click (and middle-click) are ignored and do not move the slider or start auto. This avoids the issue where right-click could previously cause the slider to move or auto-start after the context menu closed.

Browser support

The plugin uses standard DOM and JavaScript APIs and works in all modern browsers and older ones that support the following:

  • Supported: Chrome, Firefox, Safari, Edge (all current versions), IE11, IE10. Mobile browsers (iOS Safari, Chrome Android, etc.) are supported. Touch drag and mouse drag work where the browser supports them.
  • APIs used: addEventListener, classList, querySelector / querySelectorAll, requestAnimationFrame, CSS transform and transition (with vendor prefixes where needed). The script detects the correct transform prefix (WebKit, ms, Moz, O) for animations.
  • Older browsers: If the browser does not support the passive option for touch events, the plugin falls back to adding listeners without that option so it does not throw. IE9 and below are not supported (no classList or requestAnimationFrame in IE9; no transitionend).

So the plugin can be used in all browsers that meet the above (effectively IE10+ and all modern desktop and mobile browsers).

Integration into your project

Including ConnectSlider in the main project
Copy connect-slider.js into your app (e.g. webapps/Connect/js/connect-slider.js). Load it instead of jquery.bxslider.min.js. Then update the two call sites (connect.blog.js and connect.group.js) to use the native API:
  • Replace $slideQty.bxSlider(options) with ConnectSlider($slideQty[0], options) (or pass the DOM element from your list).
  • Replace $container.data('bxSlider') with the stored instance (e.g. store the return value of ConnectSlider(...) on the container: container._connectSlider = slider, or get it from the list element: listElement._connectSlider).
  • Call slider.destroySlider() as before when re-binding.
No jQuery is required for the slider itself.
